= Lab Guide: Sign and Approve Collections in Private Automation Hub

_A guide to signing and approving staged collections using Private Automation Hub's signing service._

---

[discrete]
====
*Estimated time to complete: 10 minutes*
====

In the previous module, you published two collections to Private Automation Hub. They are currently sitting in the *staging* repository, waiting for an administrator to review, sign, and approve them. In this module, you will sign and approve the collections, moving them to the *published* repository where they can be installed by users.

When you approve a collection in PAH, the hub's signing service automatically applies a GPG signature. This is different from the project signing you did earlier -- here, the hub signs the content server-side using its own GPG key, not the key you created in Module 2.

---

== Task 1: Navigate to the Approval Dashboard

Go to the `aap` tab and log in if needed.

[IMPORTANT]
====
.AAP Credentials
* **Username:** `admin`
* **Password:** `ansible123!`
====

Navigate to **Automation Content** -> **Collections** -> **Approval**. You should see both collections you published in the previous module:

* `ansible.test_collection` version `1.0.0`
* `community.lab_collection` version `1.0.0`

---

== Task 2: Sign and Approve the First Collection

Click the **Sign and approve** button next to `ansible.test_collection`. This triggers two actions:

1. The hub's signing service creates a GPG signature for the collection using the server-side signing key.
2. The collection moves from the staging repository to the published repository.

After approval, the collection disappears from the approval dashboard.

---

== Task 3: Sign and Approve the Second Collection

Click the **Sign and approve** button next to `community.lab_collection` to sign and approve it as well.

Both collections are now in the published repository and available for installation.

---

== Task 4: Verify Collections Are Published and Signed

Navigate to **Automation Content** -> **Collections**. You should see both collections listed. Look for the **Signed** indicator, which confirms that the hub's signing service has applied a cryptographic signature.

---

== Task 5: Verify Sign State via the API

You can also verify the sign state programmatically using the Galaxy API. In the `Terminal` tab, run the following commands:

[source,bash]
----
curl -u admin:ansible123! https://localhost/api/galaxy/_ui/v1/repo/published/ansible/test_collection/ -k | jq '.latest_version.sign_state'
----

[source,bash]
----
curl -u admin:ansible123! https://localhost/api/galaxy/_ui/v1/repo/published/community/lab_collection/ -k | jq '.latest_version.sign_state'
----

Both commands should return `"signed"`, confirming the collections were signed by the hub's signing service.

---

== Summary

* You signed and approved both staged collections through the PAH approval dashboard.
* The hub's signing service applied GPG signatures to the collections server-side.
* You verified the sign state using both the web UI and the Galaxy API.
